# Database Helper Documentation (`database.dart`)

This file manages the local storage for the Money Tracker app using the **SQLite** (sqflite) package. It defines the data structure, initializes the database, and performs CRUD (Create, Read, Update, Delete) operations.

---

## 1. Imports and Global Lists
- **`sqflite`**: The core package for SQLite database functionality.
- **`path`**: Utilities for joining directory paths (ensures the file path works on both Android and iOS).
- **`category` & `account`**: Global lists used by the Dropdown menus in `main.dart`. These are the "Source of Truth" for valid input types.

---

## 2. The Data Model (`TransactionModel`)
This class converts raw data into a structured Dart object.

### Fields:
- **`id`**: Unique identifier (Primary Key). It is `int?` (nullable) because new entries don't have an ID until the database assigns one.
- **`amount`**: The money value (Double).
- **`isHighlighted`**: Stored as an `int` (1 for true, 0 for false) because SQLite doesn't have a boolean type.
- **`rating`**: Importance level (1-5).

### Methods:
- **`toMap()`**: Converts the `TransactionModel` object into a `Map<String, dynamic>`. SQLite only accepts data in this Map format.

---

## 3. The Database Helper (`DatabaseHelper`)
This class handles the connection to the physical `.db` file on the phone.

- **Singleton Pattern**: Uses a private constructor and a static `_database` variable to ensure only one connection is open at a time.
- **`get database`**: A "lazy initializer" that opens the database only when needed.
- **`_initDatabase()`**:
    - Finds the safe system folder using `getDatabasesPath()`.
    - **`onCreate`**: Runs ONLY the first time the app is ever opened. It executes the SQL command to create the `transactions` table.

---

## 4. CRUD Methods

### `insertTransaction(TransactionModel transaction)`
- Opens the database.
- Uses `db.insert()` to save the map generated by `transaction.toMap()`.
- **`ConflictAlgorithm.replace`**: If an ID already exists, it overwrites the old data instead of crashing.

### `getTransactions()`
- Performs a `SELECT *` on the transactions table.
- **`List.generate`**: Converts the raw List of Maps from SQLite back into a List of `TransactionModel` objects so the UI can display them.

---

## 5. Utility Functions
- **`addCategory`**: Adds a new string to a list if it doesn't already exist (prevents duplicates).
- **`deleteCategory`**: Removes a specific string from a list.
